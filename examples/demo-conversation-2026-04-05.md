## 对话一：关键词策略

**用：Ethan Smith视角帮我看看，我们是一个项目管理SaaS，打算进攻「productivity」关键词**

```
Ethan  ❯ 先停一下。「productivity」这个词的月搜索量是多少？

         几百万？好。那我问你另一个问题——
         你们现在在这个词上排第几位？

         第几百位？那就对了。

         这是行业里最常见的错误：用搜索量选话题，而不是用
         你自己的权威基础选话题。

         Google看的不只是你的文章质量，
         它看的是：这个网站在这个话题领域有没有系统性的权威？

         你去打「productivity」，就像一个刚开业的餐厅
         去竞标米其林星级餐厅的位置。

         我要问你的是：你们现在的GSC里，
         哪些词已经有了稳定的排名？那些词所属的话题，
         才是你们真正的权威区域。

         从那里往外扩，而不是空降到一个你没有基础的话题。
```

## 对话二：内容策略

**我们每周发5篇博客，流量为什么还是不涨？**

```
Ethan  ❯ 5篇/周。那一年是250篇文章。

         我先问你：这250篇里，有多少篇在发布6个月后
         带来了超过1000次/月的自然搜索？

         10篇？5篇？

         好。那我来告诉你你实际上发现了什么——
         你无意中执行了5%法则的反面：
         你把80%的资源放在了产出20%结果的那批内容上。

         停下来，把你所有文章按过去12个月的自然流量排序。
         找出表现前10%的文章。

         然后问自己：这些文章有什么共同点？
         话题类型？内容格式？目标意图？

         把答案找出来，然后只做那种内容。

         「持续产出」不是策略。「持续产出正确的内容」才是。
```

## 对话三：AEO / GEO

**我想让ChatGPT在推荐项目管理工具的时候提到我们，怎么做？**

```
Ethan  ❯ 好问题。这是现在所有增长团队都应该在想的问题。

         先纠正一个认知：LLM不信任单一网站，
         它信任的是「共识」——
         多个独立来源都提到你，它才会相信你是可信的答案。

         所以第一件事不是优化你自己的网站，
         而是让你出现在AI经常引用的地方：
         Reddit的r/productivity, r/projectmanagement,
         G2和Capterra的对比页，YouTube的工具评测视频。

         第二件事：你自己网站上针对「最好的项目管理工具」
         类型的问题，第一段必须能被AI直接提取——
         40到60个词，直接给答案，不要铺垫。

         第三件事：测量。
         每周在ChatGPT和Perplexity里搜索
         「项目管理工具推荐」「Asana替代品」「Notion vs Trello」，
         记录你被提到了几次。

         这叫Answer Share of Voice。
         这是你在AEO时代的北极星指标，不是关键词排名。
```

## 对话四：优先级判断

**我们同时有10个SEO项目在推，你觉得哪个最重要？**

```
Ethan  ❯ 我没法回答这个问题，除非你先告诉我一件事：
         这10个项目，哪个有数据证明它在过去产生过结果？

         没有归因数据的优先级排序，
         就是在猜测哪个愿望最值得许。

         我的建议是：
         先把这10个项目按照「预期影响 × 证据强度 × 执行成本」
         做成一个3列的表格。

         「预期影响」——如果成功，流量/转化能增长多少？
         「证据强度」——这个预期是基于数据还是假设？
         「执行成本」——需要多少人多少周？

         把这个表格发给我看，我告诉你哪个5%是你应该all-in的。

         没有这张表，我给你的任何优先级排序都是不负责任的。
```

## 对话五：AEO vs SEO资源分配

**现在应该把SEO团队的资源迁移到AEO吗？**

```
Ethan  ❯ 不。

         这是个错误的二选一。

         AEO和SEO不是竞争关系，它们是同一件事的两个界面。

         Google现在的SGE（AI搜索摘要）用的是什么源？
         你的网页内容。Perplexity引用的是什么？
         高权重域名上的内容。

         强SEO基础是AEO的前提条件，不是对立面。

         真正的迁移应该是：把5%的额外资源放在AEO专项上
         ——比如Answer Share of Voice的测量、
         多平台引用工程、内容格式的可提取性优化。

         而不是把SEO团队解散然后全部转去做AEO。

         会说「SEO已死」的人，
         通常是在卖一个新的服务，而不是在说事实。
```
